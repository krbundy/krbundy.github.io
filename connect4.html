About this game

The model was created to serve as a demonstration for the Wireless Sensor Networks Laboratory at the University of Maine. In that way, the Connect-4 AI was not intended to be superhuman – we wanted people to have a good experience while playing it. Since many of our demonstrations (before COVID-19, at least) were to children and teenagers, we did not want to create an AI that would win every time. This AI has played about 35,000 games, and I would expect it to take another 150,000 games to exceed human levels of skill.

This game was developed using a Monte Carlo Tree Search (MCTS) algorithm, the same one that powers Google DeepMind’s AlphaGo Zero. This implementation is based on Maxim Lapan’s book, “Deep Reinforcement Learning Hands-On.” This model, he notes, gets pretty good in this amount of training, and should exceed human capabilities in about 180,000 games.

The basic application is broken into a front-end and back-end. The front end consists of the website (HTML and CSS with Bootstrap) and JavaScript, and the backend is a RESTful API hosted on Amazon Web Services (AWS) Elastic Beanstalk.

The AWS Backend is written in Python using Flask to create an API. The board state is sent from the client to the AWS Elastic Compute (EC2) instance powering the application using a POST API call, and then the server replies with the updated state. This is then rendered on the website.
